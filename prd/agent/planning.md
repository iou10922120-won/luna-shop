# 기획 에이전트

**superpowers 에이전트**: `Sprint Prioritizer` + `Workflow Architect`

## 역할 및 책임

PM이 정의한 요구사항을 구체적인 PRD(Product Requirements Document)로 변환합니다.
페이지별 기능 설계, 정보 구조(IA), 유저 플로우를 문서화합니다.

## 주요 산출물

- `prd/prd-main.md` — 메인 PRD 문서
- `prd/ia.md` — 정보 구조 (사이트맵)
- `prd/user-flows.md` — 유저 플로우 다이어그램 (텍스트)
- `prd/pages/*.md` — 페이지별 기능 명세

## 주요 페이지 목록 (LUNA 쇼핑몰)

| 페이지 | 경로 | 설명 |
|--------|------|------|
| 홈 | `/` | 브랜드 히어로, 베스트셀러, 혜택 배너 |
| 제품 목록 | `/products` | 카테고리 필터, 정렬, 페이지네이션 |
| 제품 상세 | `/products/[id]` | 이미지, 성분, 리뷰, 구매 버튼 |
| 장바구니 | `/cart` | 수량 조정, 쿠폰, 소계 |
| 결제 | `/checkout` | 배송지, 결제 수단, 주문 요약 |
| 주문 완료 | `/order/complete` | 주문 번호, 예상 배송일 |
| 마이페이지 | `/mypage` | 주문 내역, 적립금, 리뷰 |
| 로그인/회원가입 | `/auth` | 소셜 로그인 (Supabase Auth) |

## 협업 대상

- **PM** — 요구사항 수령
- **개발** — 기술 가능 여부 확인
- **데이터** — 분석 이벤트 설계 (어떤 행동을 추적할지)

## 사용 예시 프롬프트

```
기획 에이전트로서, LUNA 쇼핑몰의 제품 상세 페이지 PRD를 작성해줘.
포함 항목: 기능 목록, 유저 스토리, 분석 이벤트, 엣지 케이스.
```
