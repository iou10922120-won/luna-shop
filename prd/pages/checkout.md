# 결제 (/checkout)

## 기능 목록
1. 배송지 입력 (이름, 연락처, 주소)
2. 결제 수단 선택 (카드/간편결제 — MVP에서는 모킹)
3. 주문 요약 (상품 목록, 금액, 할인, 배송비)
4. 최종 결제 버튼
5. 비로그인 시 로그인/회원가입 유도

## 유저 스토리
- 구매자로서, 배송지를 입력하고 결제 수단을 선택해서 주문을 완료하고 싶다
- 재구매 유저로서, 이전에 입력한 배송지를 자동으로 불러오고 싶다
- 비로그인 유저로서, 결제 직전에 간편하게 가입/로그인하고 싶다

## 데이터 흐름

| 동작 | 읽기 (SELECT) | 쓰기 (INSERT/UPDATE) |
|------|-------------|---------------------|
| 장바구니 데이터 로드 | cart_items JOIN products WHERE user_id = ? | — |
| 이전 배송지 불러오기 | orders WHERE user_id = ? ORDER BY created_at DESC LIMIT 1 | — |
| 결제 완료 | — | orders INSERT (주문 생성) |
| — | — | order_items INSERT (주문 상품) |
| — | — | inventory UPDATE (재고 차감) |
| — | — | cart_items DELETE (장바구니 비우기) |
| — | — | sessions INSERT (purchase 이벤트) |

### 주문 생성 트랜잭션 (순서 중요)
```
1. 재고 확인 (stock_quantity >= 주문 수량)
2. orders INSERT → order_id 생성
3. order_items INSERT (각 상품)
4. inventory INSERT (sale 유형, 재고 차감)
5. products UPDATE stock_quantity (차감)
6. cart_items DELETE (해당 유저)
7. purchase 이벤트 기록
→ 하나라도 실패 시 전체 롤백
```

## 분석 이벤트

| 이벤트 | 트리거 | 속성 | 분석 연계 |
|--------|--------|------|----------|
| `page_view` | 페이지 로드 | page_path: "/checkout" | 세션 흐름 |
| `checkout_start` | 페이지 로드 | cart_total, item_count, items[] | **퍼널 4단계** |
| `purchase` | 결제 완료 버튼 클릭 (성공) | order_id, order_number, total_amount, discount_amount, shipping_fee, item_count, items[], coupon_code, payment_method | **퍼널 5단계 (매출)** |

## 분석 연계
- **퍼널 핵심**: checkout_start → purchase 전환율 = 결제 완료율 (벤치마크 70~85%)
- **결제 이탈 분석**: 결제 페이지 진입 후 미완료 세션 → 이탈 원인 추정
  - 체류 시간 분석 → 배송지 입력에서 이탈? 결제 수단에서 이탈?
- **결제수단 분포**: payment_method별 비율 → 선호 결제 수단 파악
- **매출 귀속**: purchase 이벤트의 세션 UTM → 채널별 매출 귀속
- **AOV 분석**: purchase.total_amount 분포 → 평균 주문 금액, 가격대 클러스터링

## 엣지 케이스
- 비로그인 접근: 로그인 페이지로 리다이렉트 (장바구니 유지)
- 결제 중 재고 소진: "죄송합니다, [상품명]이 품절되었습니다" → 장바구니로 복귀
- 배송지 미입력: 필수 필드 유효성 검사, 인라인 에러
- 결제 실패 (MVP 모킹): 항상 성공 처리, 실패 시나리오는 데이터에서 시뮬레이션
- 중복 결제 방지: 결제 버튼 클릭 후 로딩 상태, 재클릭 방지
