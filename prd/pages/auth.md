# 로그인/회원가입 (/auth)

## 기능 목록
1. 소셜 로그인 (Google, Kakao) — Supabase Auth
2. 회원가입 (소셜 계정 기반, 추가 정보 입력)
3. 로그아웃
4. 결제 중 가입 시 장바구니 유지 + 결제 플로우 복귀

## 유저 스토리
- 신규 방문자로서, 소셜 계정으로 간편하게 가입하고 싶다
- 비로그인 유저로서, 장바구니에 담은 상품을 유지한 채 가입/로그인하고 싶다
- 기존 유저로서, 빠르게 로그인해서 마이페이지에 접근하고 싶다

## 데이터 흐름

| 동작 | 읽기 | 쓰기 (INSERT/UPDATE) |
|------|------|---------------------|
| 소셜 로그인 | Supabase Auth (외부) | — |
| 신규 가입 | — | users INSERT (id = Supabase UID) |
| 추가 정보 입력 | — | users UPDATE (name, phone, birth_date, gender) |
| 장바구니 동기화 | 로컬스토리지 cart → | cart_items INSERT (로컬 → DB) |
| 로그인 이벤트 | — | sessions INSERT (login/signup_complete) |

### Supabase Auth 흐름
```
1. 유저가 Google/Kakao 버튼 클릭
2. Supabase OAuth redirect → 소셜 로그인 화면
3. 인증 완료 → Supabase가 auth.users에 자동 생성
4. 콜백에서 public.users 존재 여부 확인
   4a. 존재 → 로그인 완료
   4b. 미존재 → users INSERT + 추가 정보 모달
5. 로컬스토리지 장바구니 → cart_items 동기화
6. redirect_to 파라미터가 있으면 해당 페이지로 복귀
```

### UTM 처리
```
가입 시 현재 세션의 UTM 정보를 users 테이블에 저장
→ users.utm_source, utm_medium, utm_campaign
→ 가입 채널별 분석의 기준 데이터
```

## 분석 이벤트

| 이벤트 | 트리거 | 속성 | 분석 연계 |
|--------|--------|------|----------|
| `page_view` | 페이지 로드 | page_path: "/auth" | 세션 흐름 |
| `signup_start` | 소셜 로그인 버튼 클릭 (신규) | method (google/kakao) | 가입 퍼널 시작 |
| `signup_complete` | 가입 완료 (users INSERT) | method, has_profile (추가 정보 입력 여부) | 가입 퍼널 완료 |
| `login` | 로그인 완료 (기존 유저) | method | 로그인 빈도, WAU |

## 분석 연계
- **가입 퍼널**: signup_start → signup_complete 전환율 (소셜 로그인 특성상 높을 것)
- **가입 채널별 분석**: Google vs Kakao 가입 비율, 각 채널의 후속 전환율(첫 구매)
- **프로필 완성률**: 추가 정보 입력 완료 비율 → 데이터 품질 지표
- **가입 → 첫 구매 전환**: signup_complete 후 purchase까지 평균 소요 시간
- **결제 중 가입 비율**: redirect_to=/checkout인 가입의 비율 → 결제 의도 있는 가입

## 엣지 케이스
- 이미 로그인 상태에서 /auth 접근: 홈으로 리다이렉트
- 소셜 로그인 실패: "로그인에 실패했습니다. 다시 시도해주세요"
- 동일 이메일 다른 소셜: Supabase가 자동 링킹 (설정에 따라)
- 추가 정보 스킵: 가입은 완료, 이후 마이페이지에서 입력 유도
- 장바구니 동기화 충돌: 로컬 + DB 모두 있으면 합치기 (수량 합산)
