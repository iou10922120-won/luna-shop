# 마이페이지 (/mypage)

## 기능 목록
1. 유저 요약 (이름, 등급, 적립금)
2. 주문 내역 (최근 순, 상태별 필터)
3. 리뷰 관리 (작성한 리뷰 목록, 작성 가능한 주문)
4. 회원 정보 수정

## 유저 스토리
- 구매 유저로서, 내 주문 상태(배송중/완료)를 확인하고 싶다
- 구매 유저로서, 구매한 상품에 리뷰를 작성하고 싶다
- 재구매 유저로서, 이전 주문을 보고 같은 상품을 다시 구매하고 싶다

## 데이터 흐름

| 동작 | 읽기 (SELECT) | 쓰기 (INSERT/UPDATE) |
|------|-------------|---------------------|
| 유저 정보 | users WHERE id = ? | — |
| 주문 내역 | orders WHERE user_id = ? ORDER BY ordered_at DESC | — |
| 주문 상세 | order_items JOIN products WHERE order_id = ? | — |
| 리뷰 목록 (작성한) | reviews WHERE user_id = ? | — |
| 리뷰 작성 가능 목록 | order_items LEFT JOIN reviews → reviews NULL인 항목 | — |
| 리뷰 작성 | — | reviews INSERT |
| 재주문 | — | cart_items INSERT (이전 주문 상품 → 장바구니) |

### 재주문 로직
```
1. 선택한 주문의 order_items 조회
2. 각 상품의 현재 재고 확인
3. 재고 있는 상품만 cart_items에 INSERT
4. 품절 상품이 있으면 "일부 상품이 품절되어 제외되었습니다" 토스트
5. 장바구니 페이지로 이동
```

## 분석 이벤트

| 이벤트 | 트리거 | 속성 | 분석 연계 |
|--------|--------|------|----------|
| `page_view` | 페이지 로드 | page_path: "/mypage" | 세션 흐름 |
| `review_write` | 리뷰 작성 완료 | product_id, rating, content_length | 리뷰 분석, 만족도 |
| `add_to_cart` | 재주문 클릭 | product_id(s), source: "reorder" | 재구매 경로 분석 |

## 분석 연계
- **재구매 패턴**: 재주문 기능 사용률 → 재구매의 몇 %가 재주문 경로인지
- **리뷰 작성률**: 구매 대비 리뷰 작성 비율, 작성까지 평균 소요 일수
- **평점 분포**: 상품별 평점 분포 → 만족도 분석, 개선 필요 상품 식별
- **리뷰와 재구매 상관**: 리뷰 작성 유저의 재구매율 vs 미작성 유저 비교
- **마이페이지 방문 빈도**: 자주 방문하는 유저 = 충성 고객 시그널

## 엣지 케이스
- 주문 내역 없음: "아직 주문 내역이 없어요" + [쇼핑하러 가기]
- 리뷰 이미 작성: [리뷰 완료] 비활성 상태
- 리뷰 내용 미입력: 평점만 필수, 내용은 선택
- 재주문 상품 전부 품절: "해당 주문의 모든 상품이 품절입니다" 알림
- 비로그인 접근: /auth로 리다이렉트
