# 주문 완료 (/order/complete)

## 기능 목록
1. 주문 완료 확인 메시지
2. 주문 번호 및 주문 요약
3. 예상 배송일 표시
4. 쇼핑 계속하기 / 주문 내역 바로가기

## 유저 스토리
- 구매 완료한 유저로서, 주문이 정상적으로 처리되었는지 확인하고 싶다
- 구매 완료한 유저로서, 언제 배송되는지 알고 싶다

## 데이터 흐름

| 동작 | 읽기 (SELECT) | 쓰기 |
|------|-------------|------|
| 주문 정보 로드 | orders JOIN order_items JOIN products WHERE order_id = ? | — |
| 페이지 진입 | — | sessions (page_view) |

### 예상 배송일 로직
- 기본: 주문일 + 2영업일
- MVP에서는 단순 계산 (주말 제외하지 않음, +2일)

## 분석 이벤트

| 이벤트 | 트리거 | 속성 | 분석 연계 |
|--------|--------|------|----------|
| `page_view` | 페이지 로드 | page_path: "/order/complete", order_id | 퍼널 완료 확인 |

주: `purchase` 이벤트는 /checkout에서 이미 발생. 이 페이지는 확인 페이지일 뿐.

## 분석 연계
- **퍼널 완료 확인**: 이 페이지 도달 = 구매 완료. purchase 이벤트와 교차 검증
- **후속 행동 분석**: 주문 완료 후 "쇼핑 계속하기" vs "주문 내역" 클릭 비율 → 추가 구매 가능성
- **이 페이지 자체의 분석 가치는 낮음** — 핵심 분석은 checkout의 purchase 이벤트에서 발생

## 엣지 케이스
- 직접 URL 접근 (주문 없이): "주문 정보를 찾을 수 없습니다" → 홈으로 리다이렉트
- 새로고침: 동일 정보 표시 (멱등성 보장)
- 이전 주문 조회: order_id 파라미터로 이전 주문도 표시 가능
